\subsection{Purpose of V\&V in this framework}
Verification and validation are first-class requirements. Verification confirms that the framework is implemented and executed according to specification. Validation confirms that outputs are meaningful for interpreting knowledge flow.

\subsection{Verification protocol}
A framework execution is verified only when all of the following checks are satisfied:
\begin{enumerate}
    \item \textbf{Data integrity checks}: identifiers, metadata, and citation edges are internally consistent and auditable.
    \item \textbf{Temporal consistency checks}: parent-child direction respects publication chronology.
    \item \textbf{Embedding consistency checks}: vector norms, duplicate handling, and representational diagnostics are stable.
    \item \textbf{Optimization checks}: solver convergence, constraint feasibility, and coefficient validity are confirmed.
    \item \textbf{Reproducibility checks}: repeated runs under fixed configuration reproduce equivalent outputs within accepted numeric tolerance.
\end{enumerate}

\subsection{Validation protocol}
Validation is performed with triangulated evidence rather than a single benchmark label:
\begin{enumerate}
    \item \textbf{Reconstruction fidelity}: evaluate the degree to which parent mixtures explain child embeddings.
    \item \textbf{Stability under perturbation}: re-estimate inheritance under parent-set resampling to characterize uncertainty.
    \item \textbf{Interpretive plausibility}: inspect high- and low-contribution cases for consistency with domain expectations.
    \item \textbf{Divergence profiling}: compare contribution rankings with popularity-oriented baselines to detect distinct signal content.
\end{enumerate}

\subsection{Acceptance criteria for publishable use}
Outputs are considered publication-ready only if:
\begin{itemize}
    \item verification checks pass,
    \item uncertainty is reported for non-robust estimates,
    \item interpretive claims remain consistent with limitations,
    \item and analyses are reproducible from documented inputs and parameters.
\end{itemize}

\subsection{Failure modes to disclose}
Any report using this framework must disclose major risks that can alter interpretation:
\begin{itemize}
    \item missing or noisy citation edges,
    \item embedding degradation for domain-specific language,
    \item collinearity among parents,
    \item and instability of weak inheritance edges.
\end{itemize}

This disclosure requirement is part of the framework definition, not an optional reporting preference.
